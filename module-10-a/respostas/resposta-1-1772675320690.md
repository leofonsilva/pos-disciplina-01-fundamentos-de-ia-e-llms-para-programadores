Para converter objetos JavaScript em tensores no TensorFlow.js, você precisa transformar seus dados em uma estrutura numérica que o TensorFlow.js possa entender. Vou explicar isso com base no exemplo do contexto.

**O problema:** O TensorFlow.js só trabalha com números, então objetos JavaScript com propriedades como strings (nomes, cores, localizações) precisam ser convertidos para representações numéricas.

**A solução - One-Hot Encoding:** No exemplo das pessoas, as cores preferidas foram convertidas usando uma técnica chamada *one-hot encoding*. Veja como funciona:

```
azul vermelho verde
Erick   1     0      0
Ana     0     1      0
Carlos  0     0      1
```

Cada cor se torna uma coluna, e a cor preferida de cada pessoa recebe o valor 1, enquanto as outras recebem 0.

**Como fazer isso na prática:**

1. **Para propriedades numéricas** (como idade), você pode usar os valores diretamente:
```javascript
const idades = pessoas.map(p => p.idade); // [30, 25, 40]
```

2. **Para propriedades categóricas** (como cor), crie uma matriz one-hot:
```javascript
// Mapear cores para índices
const cores = ['azul', 'vermelho', 'verde'];
const corParaIndice = {};
cores.forEach((cor, i) => corParaIndice[cor] = i);

// Criar matriz one-hot
const coresOneHot = pessoas.map(p => {
  const linha = new Array(cores.length).fill(0);
  linha[corParaIndice[p.cor]] = 1;
  return linha;
});
```

3. **Combinar tudo em um tensor:**
```javascript
const tensorDados = tf.tensor2d([
  ...pessoas.map(p => [p.idade, ...coresOneHot.find(c => c.includes(1))])
]);
```

**Analogia prática:** Imagine que você está organizando uma festa e quer categorizar os convidados por preferência musical. Em vez de dizer "rock", "samba" ou "eletrônica", você cria colunas: rock=1/0, samba=1/0, eletrônica=1/0. Isso é exatamente o que fazemos com one-hot encoding!

O resultado final é um tensor numérico que o TensorFlow.js pode processar para treinar modelos de machine learning.
